# 常见问题处理

## 问题1：图片中出现乱码文字

**场景**：生成的图片中出现无意义的文字或字母

**解决方案**：

1. **强化提示词**：在提示词末尾添加多重否定
   ```
   no text, no letters, no words, no writing, no captions, text-free
   ```

2. **避免触发文字的元素**：
   - 避免描述"书""海报""标志""屏幕"等容易出现文字的物体
   - 如必须包含，明确指定：`blank book cover`, `empty screen`

3. **重新生成**：如仍有文字，直接重新生成，AI生图有随机性

---

## 问题2：多张图片中人物外观不一致

**场景**：同一篇文章中，同一角色在不同配图中长相差异明显

**解决方案**：

1. **创建角色卡片**：首张图生成后，记录详细外观描述
   ```
   角色锚定：young Asian woman, age 30, short black bob hair, 
   wearing navy blue blazer, round silver glasses, warm smile
   ```

2. **每张图复用相同描述**：将角色描述作为固定前缀

3. **降低期望**：当前AI生图技术对人物一致性支持有限，建议：
   - 接受轻微差异（表情、角度变化）
   - 保持服装、发型、标志性特征一致即可
   - 如要求极高一致性，考虑使用同一张人物图 + 不同背景合成

4. **风格选择**：扁平插画(flat)和卡通(cartoon)风格的人物一致性通常比写实风格更容易控制

## 问题3：段落内容过于抽象，难以生成配图

**场景**：段落讨论抽象概念如"用户增长策略""商业模式创新"

**解决方案**：

| 抽象概念 | 转化为具象画面 |
|----------|----------------|
| 用户增长 | 上升的曲线图、人群聚集、阶梯向上 |
| 商业模式 | 齿轮运转、拼图组合、建筑蓝图 |
| 团队协作 | 多人围坐讨论、接力赛跑、乐队演奏 |
| 创新突破 | 灯泡亮起、破壳而出、火箭发射 |
| 数据分析 | 图表仪表盘、放大镜观察、数字流动 |
| 时间紧迫 | 沙漏、时钟、奔跑的人 |

**提示词模板**：
```
Abstract concept of [概念] visualized as [具象画面], 
metaphorical representation, symbolic illustration
```

## 问题4：生成的图片风格不一致

**原因**：每次生成时提示词风格关键词有差异

**解决方案**：

1. **创建风格锚定词**：在每个提示词末尾固定追加相同的风格描述
   ```
   [具体内容描述], consistent style with previous images, 
   flat illustration, warm color palette, minimal design
   ```

2. **使用种子值**（如工具支持）：固定随机种子保持风格一致

3. **批量检查**：生成后对比所有图片，风格偏离明显的重新生成

---

## 问题5：图片与段落内容不相关

**原因**：提示词未准确捕捉段落核心

**解决方案**：

1. **提取核心名词**：识别段落中最具画面感的1-2个名词
2. **确认动作/状态**：明确主体在做什么或处于什么状态
3. **添加情感氛围**：补充段落的情感基调

**检查清单**：
- [ ] 提示词包含段落的核心主体？
- [ ] 提示词描述了具体场景而非抽象概念？
- [ ] 提示词的情感氛围与段落匹配？

---

## 问题6：用户无法使用图像生成API

**场景**：用户没有DALL-E/MJ等工具的API访问权限

**替代方案**：

1. **输出提示词清单**：提供可直接复制到MJ/SD的提示词
2. **推荐免费工具**：
   - Bing Image Creator（免费，基于DALL-E）
   - Leonardo.ai（每日免费额度）
   - Playground AI（每日免费额度）
3. **提供Canva/Figma建议**：根据提示词描述，建议可搜索的素材关键词

---

## 问题7：配图数量过多/过少

**过多**：
- 合并相邻短段落
- 调整为"低密度"配图策略
- 仅为核心论点段落配图

**过少**：
- 拆分过长段落
- 调整为"高密度"配图策略
- 在语义转换点增加配图

**快速调整**：
```
用户反馈后，调整配图密度参数重新规划，无需重新分析全文
```

## 问题8：特定领域内容配图困难

### 技术/代码类
- 不为代码块直接配图
- 可为"技术概念说明"段落配架构图风格插画
- 提示词示例：`technical diagram showing [概念], blueprint style, clean lines`

### 财经/数据类
- 避免生成具体数字的图表（AI生成的数字通常不准确）
- 生成氛围图：股市大屏、交易场景、数据可视化风格背景
- 提示词示例：`abstract data visualization, financial theme, professional`

### 法律/政策类
- 使用象征性图像：天平、法槌、文件签署
- 保持严肃专业风格
- 提示词示例：`legal concept illustration, professional, minimal, scales of justice`

## 问题9：图片尺寸/比例不符合平台要求

**快速参考**：

| 平台 | 封面比例 | 内文图比例 | 提示词参数 |
|------|----------|------------|------------|
| 公众号 | 2.35:1 | 任意（建议16:9） | --ar 2.35:1 |
| 小红书 | 3:4 或 1:1 | 3:4 | --ar 3:4 |
| 知乎 | 16:9 | 任意 | --ar 16:9 |
| 头条 | 16:9 | 任意 | --ar 16:9 |

**批量调整**：如需统一调整已生成图片比例，建议使用Canva/Figma裁剪而非重新生成
