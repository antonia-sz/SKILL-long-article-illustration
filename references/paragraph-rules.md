# 段落划分规则

## 划分原则

### 优先级顺序

1. **显式结构标记**（最高优先级）
2. **语义边界识别**
3. **长度均衡调整**（最低优先级）

---

## 规则详解

### 规则1：显式结构标记

识别以下标题/分隔标记，作为段落边界：

| 标记类型 | 示例 | 处理方式 |
|----------|------|----------|
| Markdown标题 | `#` `##` `###` | 每个标题开启新段落 |
| 数字序号 | `1.` `2.` `（1）` `（一）` `第一章` | 每个序号开启新段落 |
| 中文序号 | `一、` `二、` `首先` `其次` | 每个序号开启新段落 |
| Emoji/特殊序号 | `1️⃣` `2️⃣` `①` `②` `❶` `❷` | 每个序号开启新段落 |
| 分隔线 | `---` `***` `===` | 分隔线前后各为一段 |
| 空行分隔 | 连续2个以上空行 | 视为段落分隔 |

### 规则2：语义边界识别

当无显式标记时，识别以下语义转换点：

| 边界类型 | 识别信号 | 示例 |
|----------|----------|------|
| **时间跳跃** | 时间词变化 | "三年前..." → "如今..." |
| **场景切换** | 地点/环境变化 | "在办公室..." → "回到家后..." |
| **主题转换** | 讨论对象变化 | 从产品功能 → 用户反馈 |
| **视角转换** | 叙述角度变化 | 从个人经历 → 行业观察 |
| **论证层次** | 论点推进 | 提出问题 → 分析原因 → 给出方案 |

**语义边界信号词**：
```
转折：但是、然而、不过、可是、事实上
递进：而且、更重要的是、不仅如此、此外
因果：因此、所以、由此可见、正因为如此
转换：另一方面、与此同时、说到这里、话说回来
总结：总之、综上所述、总的来说、最后
```

### 规则3：长度均衡调整

在前两条规则基础上进行微调：

| 情况 | 处理方式 |
|------|----------|
| 段落过长（>800字） | 在语义次边界处拆分 |
| 段落过短（<150字） | 与相邻段落合并（主题相关时） |
| 配图密度要求 | 根据用户设定的配图密度调整 |

**理想段落长度**：300-600字/段（对应一张配图）

---

## 配图密度策略

| 密度设置 | 说明 | 适用场景 |
|----------|------|----------|
| **高密度** | 每段一图 | 教程、图文并茂的科普 |
| **中密度** | 每2-3段一图（默认） | 一般文章、散文 |
| **低密度** | 每小节一图 | 长篇深度文章、学术内容 |
| **关键点** | 仅核心段落配图 | 商务报告、严肃内容 |

---

## 段落输出格式

```markdown
---
【段落 1】{段落标题或摘要}
配图建议：✅ 需要配图 / ⏭️ 可跳过

{段落正文内容}

---
【段落 2】{段落标题或摘要}
配图建议：✅ 需要配图

{段落正文内容}

---
```

---

## 不适合配图的段落

以下类型段落建议跳过配图：

| 类型 | 原因 |
|------|------|
| 纯数据/表格 | 数据本身即为视觉元素 |
| 引用块 | 通常用于引述，配图可能干扰 |
| 代码块 | 技术内容，图片无附加价值 |
| 过渡段 | 承上启下，内容较空 |
| 结尾致谢 | 非正文内容 |

---

## 特殊处理

### 列表段落
- 短列表（<5项）：整体作为一段，生成一张综合配图
- 长列表（≥5项）：考虑是否需要配图，或生成信息图风格

### 对话/问答
- 连续对话视为一个段落
- 配图侧重场景氛围而非具体对话内容

### 故事/叙事
- 按场景切换划分
- 配图侧重画面感最强的瞬间
